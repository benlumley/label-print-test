Arvensis Label Printer Test

How to use this page (read first)

Setup

  1. Install QZ Tray on the machine the printer is connected to (free, Mac/Win/Linux). Launch it — you should see its icon in your system tray.
  2. The first time you click Connect, your browser may block the websocket because QZ Tray uses a self-signed cert on localhost:8181. If that happens, open https://localhost:8181 in a new tab, accept the certificate warning, then come back and try Connect again.
  3. This page does not sign requests, so QZ Tray will pop an “Allow this site to print?” dialog. Click Allow (and tick “remember” to stop being asked).

Sending a test print

  1. Click Connect — the dot should turn green.
  2. Click Find Printers — click your label printer in the list to select it.
  3. Pick the Format that matches your printer’s command language:
    • Zebra → ZPL
    • Toshiba TEC (e.g. SA4) → TPCL (or ZPL if it’s in ZPL emulation mode)
    • Eltron / older Zebra → EPL2
    • TSC / Printronix → TSPL
    • Receipt printers → ESC/POS
  4. If you’re unsure of the language, try Raw Text first — that confirms the pipe works end-to-end before you fight with command syntax.
  5. Hit Send Test Print. Status appears below the button.

If it doesn’t work

  • Connect fails → QZ Tray isn’t running, or browser is blocking the websocket. Check the tray icon, then visit https://localhost:8181 and accept the cert.
  • “Sent successfully” but nothing prints → usually wrong format for the printer, or the driver is mangling raw bytes. Try Raw Text to isolate; if that prints, it’s a format mismatch.
  • Prints garbled characters / command codes as text → printer is treating commands as plain text. Wrong format selected, or printer is in the wrong emulation mode.
  • Connection drops every print → expected on community QZ Tray; the “Allow” dialog reconfirms each session.
